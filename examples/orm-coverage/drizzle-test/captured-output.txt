Generated migration file content :

  	"email" varchar(255) NOT NULL

================================================================

Note that the schema in TypeScript declared THREE fields (id,
email, emailAddress) but the generated SQL contains only TWO
columns. The `emailAddress` field is silently absent from the
migration output.

This is the failure mode the canonical Typerion case-01 fixture
describes : two TypeScript fields whose SQL projection collapses
onto one column. Drizzle does not warn ; the second field is
simply not present in the generated database schema.

A runtime write to `user.emailAddress` would attempt to write to
a column that the migration never created.
